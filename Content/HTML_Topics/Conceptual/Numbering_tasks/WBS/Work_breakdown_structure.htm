Work breakdown structure

Een work breakdown structure (WBS) is een resource dat wordt gebruikt om de afzonderlijke werkonderdelen van een project (de taken van een project) te definiëren en te groeperen op een manier die helpt de totale werklast van het project te organiseren en te definiëren. Elk element binnen een work breakdown structure element kan staan voor een product, data, of dienst, of een combinatie van bovenstaande. Een WBS voorziet ook in een raamwerk voor gedetailleerde kostenraming en controles.

Een work breakdown structure is een boomstructuur waarin alle elementen op een niveau worden samengevat door elementen in het bovenliggende niveau. Een WBS is nuttig wanneer u het werk dat komt kijken bij een project onder wilt verdelen, beginnend met de hoofdfases van het project en uitbreidend naar de individuele taken die moeten worden uitgevoerd. Ieder element binnen een WBS heeft een eigen alfanumerieke WBS code en een beschrijvende WBS naam.

Om te kunnen werken met een work breakdown structure zet u eerst de hiërarchie van WBS codes in een project op. U kunt dan WBS codes toewijzen aan de taken in het project, door het Selecteer WBS Code dialoogvenster te gebruiken, of door de WBS codes vanuit de project view aan te klikken en naar de taken te slepen, wanneer de WBS hiërarchie getoond kan worden.

Wanneer u WBS codes heeft toegewezen aan de taken in een project kunt u de taken sorteren/groeperen op WBS code. Hiermee heeft u een andere mogelijkheid om de structuur van uw project te bekijken: taken worden getoond volgens hun positie in de WBS hiërarchie, in plaats van volgens hun positie in de projecthiërarchie. U kunt taken ook filteren op de WBS codes die er aan toegewezen zijn.
