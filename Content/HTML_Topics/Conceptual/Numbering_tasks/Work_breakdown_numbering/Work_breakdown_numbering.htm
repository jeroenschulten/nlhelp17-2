WBN Taak ID Nummering

Taken in een WBS kunnen een uniek nummer krijgen om hun positie in de hiërarchie te identificeren. Dit wordt taak ID (WBN) nummering genoemd. U kunt taak ID (WBN) codes voor taken in een project genereren.

Het kan een vereiste zijn om taak ID (WBN) codes te gebruiken in plaats van balk of taaknamen om te voldoen aan een bepaalde bedrijfsstandaard. Sommige bedrijven prefereren het gebruik van taak ID (WBN) codes, omdat ingesteld kan worden dat deze altijd uniek zijn binnen het gehele project, terwijl balk- en taaknamen dat misschien niet altijd zijn. Ook is het mogelijk om te filteren op taak ID (WBN) codes waardoor ze een handig zoekinstrument vormen.

Als u taken toevoegt aan een Uitgeboekt bestand en u gebruikt taak ID (WBN) codes om de taken te nummeren, dan kan het voorkomen dat de taken in het uitgeboekte project taak ID (WBN) nummers krijgen die ook al gegeven zijn aan andere taken in het master project. Als dat het geval is, dan zullen deze taken conflicteren als u het doelproject weer inboekt in het master project. Om dit te vermijden kunt u de taak ID (WBN) nummering uitzetten in het uitgeboekte project.

Wanneer u balken en taken vanuit het ene project naar een andere kopieert, blijft de taak ID (WBN)-nummering behouden in de gekopieerde taken, behalve als één van de taak ID (WBN)-nummers conflicteert met de nummers in het bestemmingsproject.

WBN taak ID nummering was tot op heden in in gebruik om taken in een soort van work breakdown structure (WBS) te coderen. Wanneer het uw bedoeling is om een nette work breakdown structure in een project op te zetten is het aan te bevelen om 's work breakdown structure functie te gebruiken in plaats van WBN taak ID nummering.
