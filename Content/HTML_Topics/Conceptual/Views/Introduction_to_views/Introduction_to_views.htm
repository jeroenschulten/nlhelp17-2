Introductie over views

Een view is een venster op uw project. Een view kan bijvoorbeeld weergeven: de programma kaart, een subkaart, taken uit een aantal kaarten, alle taken met toewijzingen van een bepaalde resource of elke taak in het gehele projecttaken uit een aantal samenvattingsgroepen, of alle taken met toewijzingen van een bepaalde resource. In een bepaalde view kan de project view uitgeschakeld zijn, kunnen bepaalde kolommen worden weergegeven in de tabel en kunnen bepaalde balkkaart karakteristieken aan- of uitgezet zijn. Elke view wordt weergegeven in een tabblad - of in een venster, wanneer u heeft geconfigureerd om views in aparte vensters in plaats van in tabbladen weer te geven.

Een view kan iedere subset van projectgegevens weergeven en die presenteren met elke combinatie van weergaveopties.

Als u een view opslaat, worden de volgende instellingen opgeslagen:

  • De opmaak van de balkkaart.
  • De tabel.
  • Het actieve filter en de actieve sortering/groepering.
  • De positie van de scheidslijnen in het balkkaartvenster.
  • Het bereik: welke kaarten zijn geselecteerd in de project view (en worden dus weergegeven in de view).

Merk op dat een view verbonden zou kunnen zijn aan meer dan één type filter: een balk en taak filter, voor het filteren op de projecthiërarchie, en een RBS/CBS filter, voor het filteren op RBS en CBS views. U kunt ook een standaard afdrukprofiel toepassen op elke view, zodat hetzelfde afdrukprofiel altijd standaard wordt gebruikt wanneer een u bepaalde view afdrukt.

Wanneer u een project opslaat worden alle veranderingen die u heeft aangebracht bij alle op dit moment geopende views opgeslagen. Voorbeeld: als u in een open view de rasterlijnen in de kalenderschaal verandert en u slaat het project op dan worden de huidige rasterlijnen opgeslagen in de view. Andere vensters die dezelfde view delen worden niet up to date gebracht totdat ze gesloten en heropend worden.

De Standaard view

Elk project heeft een standaard balkkaart view, die de interne view standaards hanteert. Als u een nieuw project start, wordt automatisch de standaard balkkaart view gebruikt. Als u een project voor het eerst opslaat, wordt de huidige view opgeslagen als balkkaart view. Elke gebruiker heeft zijn/haar eigen balkkaart view, dus als u een project opslaat, worden uw instellingen met de view opgeslagen.

U hoeft niet meer views aan te maken, tenzij u dat wenst; U kunt volledig werken met uw standaard balkkaart view zonder ooit andere views aan te maken.

Als u een project opent, wordt de view geladen, die actief was, toen u het project voor het laatst heeft opgeslagen. U kunt nog altijd de standaard balkkaart view laden door in het tabblad View, in de View Gegevens groep op het View keuzelijstje te klikken en vervolgens Standaard Balkkaart View te selecteren.

Wanneer u in een Enterprise omgeving gebruikt, kunt voor elke gebruiker een standaard view specificeren. Wanneer u dit doet wordt elke standaard view standaard mee geopend wanneer een gebruiker een project opent. U kunt ook een standaard view specificeren voor het project zelf, en specificeren dat deze view geopend moet worden wanneer een gebruiker (wie dan ook) het project opent.

 Project views en gebruiker views

Er zijn twee type views in Asta :

  • Project views - views die zijn gedefinieerd door een gebruiker die de rechten heeft om zulke views te genereren. Project views zijn toegankelijk voor alle gebruikers.
  • Gebruikersviews - views die zijn gecreëerd door gebruikers, die niet recht hebben om een project view te genereren. U kunt opgeven of gebruikersviews al dan niet toegankelijk zijn voor andere gebruikers

U geeft aan of een gebruiker project views kan aanmaken in het dialoogvenster Gebruikers Eigenschappen. U kunt er voor kiezen om het recht om project views te genereren beperken tot een onderafdeling van uw bedrijf en de project views te zien als de "officiële" views.

U geeft aan of gebruiker views beschikbaar zijn voor andere gebruikers in het dialoogvenster View Eigenschappen. Gebruiker views die beschikbaar zijn voor alle gebruikers werken op een manier die erg lijkt op die van project views, maar ze hebben niet de 'officiële' status van project views.

Elk project heeft een standaard balkkaart view, die de interne view standaardinstellingen gebruikt. Wanneer u een nieuw project start, wordt de standaard balkkaart view automatisch gebruikt. Alle wijzigingen die u aanbrengt in de standaard balkkaart view, de enige beschikbare view tenzij een project is geopend in , worden automatisch opgeslagen.

In aanvulling op het type view dat hierboven beschreven wordt - balkkaart views - is er ook een ander type view - kalender views.
