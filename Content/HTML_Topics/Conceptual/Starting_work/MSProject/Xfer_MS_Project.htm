Overzetten van gegevens tussen en Microsoft Project

U kunt uw projecten beschikbaar stellen voor Microsoft Project gebruikers door projecten te exporteren naar Microsoft Project, en u kunt projecten openen die in Microsoft Project zijn gemaakt door deze te importeren in . Hiermee kunt u project informatie delen met collega's die wel toegang hebben tot Microsoft Project maar niet tot .

kan exporteren naar en importeren van twee verschillende door Microsoft Project gebruikte bestandsformaten:

  • XML - gebruikt door Microsoft Project vanaf versie 2002.
  • MPP - Microsoft Project's eigen bestandsformaat. Merk op dat u alleen kunt exporteren en importeren vanuit dit bestand wanneer u Microsoft Project op dezelfde computer heeft geïnstalleerd als .

Wanneer u Microsoft Project op dezelfde computer heeft geïnstalleerd, kunt u er ook voor kiezen om alleen de huidige groep of view naar het XML of MPP formaat te exporteren. Ook kunt u er dan voor kiezen om als MPP or XML bestanden opgeslagen Microsoft Project projecten in specifieke kaarten of samenvattingsgroepen binnen al aanwezige projecten te importeren.

Verschillen tussen en Microsoft Project bestandsformaten

Door de beperkingen van XML en MPP bestandsformaten kan bepaalde informatie van verloren gaan bij het exporteren naar één van deze formaten. Overeenkomend, kan bepaalde informatie in XML of MPP bestanden veranderen of verloren gaan bij het importeren naar . Om deze reden, wordt geadviseerd om wanneer XML of MPP bestanden geïmporteerd zijn in , het project meteen in bestandsformaat op te slaan, - in PP bestand - ter voorkoming van het verliezen van informatie die niet door XML en MPP formaten wordt ondersteund.

Bestandsverschillen bij het exporteren van projecten naar MPP of XML formaat

Let bij het exporteren van een bestand naar een XML bestand op de volgende punten.

Balken

Microsoft Project ondersteunt geen balken, daarmee ondersteunt ze ook 's subtitels -balken met een naam maar zonder taken- niet. Bij het exporteren kunt u ervoor kiezen om subtitels te negeren of om deze om te zetten in verborgen taken die de naam hebben van de balk. Let op, wanneer u voor deze optie kiest zullen deze taken die de subtitel weergeven, ondanks dat ze niet zichtbaar zijn in Microsoft Project, de samenvattingstaak beïnvloeden.

Omdat Microsoft Project geen balken ondersteunt, zullen alle balk-niveau gebruikersvelden verloren gaan tijdens het exporteren.

Taken die in op dezelfde balkregel staan, staan na de conversie op verschillende regels.

Microsoft Project

Mijlpalen

ondersteunt start- en eind mijlpalen waar Microsoft Project maar één type mijlpaal ondersteunt. Hierdoor worden alle mijlpalen bij conversie omgezet naar hetzelfde type mijlpaal.

Niveautaken

Microsoft Project ondersteunt geen niveautaken. Alle niveautaken worden omgezet in samenvattingstaken bij het exporteren.

Samenvattingstaken

Microsoft Project ondersteunt geen lege samenvattingstaken. Alle lege samenvattingstaken worden omgezet in normale taken bij het exporteren.

Hammock taken

Microsoft Project ondersteunt geen hammock taken. Alle hammock taken raken verloren bij het exporteren.

Buffertaken

Microsoft Project ondersteunt geen buffertaken. Alle buffertaken worden bij het exporteren omgezet in normale taken, met bijbehorende deadline vlag, maar worden wel gemerkt als buffertaken door het gebruik van een extra veld zodat ze weer naar buffertaken worden geconverteerd wanneer de bestanden weer in geïmporteerd worden. Let op dat geen extra beperkingen toegepast zijn op buffertaken die geen inkomende link hebben (aangezien Microsoft Project niet meer dan één link bij een taak ondersteunt). Buffertaken zonder inkomende links worden verplaatst naar de begindatum van het project, tenzij er een voorafgaande taak op dezelfde balk staat.

Duur berekende taken

Microsoft Project ondersteunt geen duur berekende taken. Alle duur berekende taken worden omgezet naar normale taken tijdens het exporteren, waarbij de huidige duur in plaats van de oorspronkelijke duur wordt gebruikt.

Task activities

Microsoft Project ondersteunt geen taakacties. Alle taakactie-informatie wordt verwijderd van de taken bij het exporteren.

Links met positieve/negatieve wachttijd

Microsoft Project ondersteunt alleen een enkele positieve/negatieve wachttijd aan één van de uiteinden van een link. Wanneer een link meer dan één wachttijd heeft worden deze twee bij het exporteren gecombineerd in een enkele waarde.

Links met positieve/negatieve wachttijd in percentages worden omgezet naar hun equivalent, tenzij de links meer dan één wachttijd hebben. In dat laatste geval worden de positieve/negatieve waarden omgezet in op tijd gebaseerde waarden die gecombineerd wordt tot 1 waarde.

Kalenders

Kalenders zijn in Microsoft Project minder flexibel dan in : ze moeten bestaan uit zeven terugkerende dagen met maximaal 5 werktijd periodes. Objecten in een project die een kalender gebruiken die niet overeenkomt met deze specificaties, worden omgezet naar een standaard kalender bij de conversie.

Microsoft Project staat niet toe dat er kalenders zijn met dezelfde naam, zodat wanneer twee kalenders dezelfde naam hebben in een project, de namen van alle behalve de eerste kalender een beetje aangepast worden tijdens het exporteren. De "Kalenders" map zelf wordt niet geëxporteerd.

Microsoft Project heeft een kalendernaamlimiet van 52 tekens. Als een project kalenders bevat met namen die langer zijn dan 52 tekens, worden deze namen tijdens het exportproces afgekapt.

Microsoft Project ondersteunt maar twee soorten uitzonderingen: werkend en niet-werkend. Alle uitzonderingen, ondanks hun naam, moeten omgezet worden naar één van deze twee soorten uitzonderingen. Bij het exporteren worden ze omgezet naar de relevante uitzondering.

Kosten en inkomsten tarieven

De kosten van resources worden in Microsoft Project anders berekend. Kosten en inkomsten die weergegeven worden bij resources zullen verdwijnen bij de omzetting, resource kosten worden berekend aan de hand van tarieven van de toewijzingen. Meerdere kost- en inkomsten-tarieven per toewijzingen worden bij het exporteren gecombineerd tot één enkele waarde.

Resources in Microsoft Project kunnen uit maximaal 5 tarieven bestaan, met daarin kosten die kunnen veranderen over de tijd: tot vijfentwintig keer in totaal. Om met deze beperkingen van de tarieven van Microsoft Project te kunnen werken, worden de resources gedupliceerd (met toevoegingen als .2, .3, enz.) bij de export, als er meer dan vijf toewijzingen van dezelfde resource op dezelfde dag met verschillende tarieven zijn, of wanneer het tarief van een toewijzing meer dan vijfentwintig keer per dag veranderd. Let op: de waarden voor de kosten van deze resources zullen niet correct zijn.

Resources en kosten

Microsoft Project ondersteunt slechts één toewijzing per resource aan een taak, wat onvoldoende is om meervoudige resourcetoewijzingen weer te geven die mogelijk zijn in . Als een taak in meer dan één toewijzing van dezelfde resource heeft worden bij het exporteren de resources gedupliceerd (met toevoegingen als .2, .3 enz.). U kunt kiezen om al dan niet de resource- en kostentoewijzingen te exporteren.

Negatieve kosten en inkomsten

ondersteunt positieve en negatieve kosten, en positieve en negatieve inkomsten. Microsoft Project ondersteunt positieve en negatieve kosten, maar kent het concept inkomsten niet. U specificeert hoe inkomsten moeten worden behandeld tijdens het exporteerproces. U kunt ervoor kiezen om alle positieve inkomsten te converteren naar negatieve kosten en alle negatieve inkomsten naar positieve kosten in het resulterende bestand, of om alle inkomsten helemaal te verwijderen in het resulterende bestand.

Overwerk

Overwerk in is onverenigbaar met Microsoft Project wanneer er sprake is van voortgang. Alle het overwerk wordt omgezet in werktijd bij het exporteren. Ondanks dat de gerelateerde kosten hetzelfde blijven na de export, is de werktijd niet te herkennen als overwerk.

Variabele datums

Microsoft Project ondersteunt geen variabele datums. Wanneer u een project exporteert naar Microsoft Project formaat worden alle variabele datums binnen het project omgezet in vaste datums en worden die gebaseerd op de datums waarin de variabele datums resulteren op de dag waarop u het project opslaat. Bijvoorbeeld, wanneer u een project naar Microsoft Project formaat exporteert op 23/11/2016, wordt de variabele datum van 'Vandaag' omgezet in de vaste datum 23/11/2016.

Codebibliotheken

Codebibliotheek toewijzingen bij elke taak en resource kunnen worden geëxporteerd naar een extra veld van het "tekst" type, met de naam "CodeLibrary:<Codebibliotheek item naam>". Bijvoorbeeld, Wanneer een "Onderaannemer" codebibliotheek item is toegewezen aan een taak, wordt er een extra veld met de naam "CodeLibrary:Onderaannemer" gecreëerd bij de taak tijdens de export. Codebibliotheek opmaak gegevens worden helemaal niet geëxporteerd.

Work breakdown structure

Wanneer er een work breakdown structure gedefinieerd is in een project, wordt de positie van elke taak in de work breakdown structure geëxporteerd in een extra veld van het "outline code" type. Wanneer er een work breakdown structure in een project bestaat, worden de items van de WBS hiërarchie direct gemapt in het eerste outline code extra veld voor elke taak, zodat de WBS code geëxporteerd wordt als de outline code waarde en de WBS naam geëxporteerd wordt als de outline code omschrijving, zoals hieronder geïllustreerd wordt:

Standaard definieert Microsoft Project het punt (full-stop) teken als haar scheidingsteken. Wanneer één van de WBS codes punttekens bevat worden deze tekens tijdens het exporteren verwijderd; wanneer dit zou resulteren in een lege WBS code, wordt er een vraagteken ingevoegd als WBS code. Daarom is het, wanneer u een project naar Microsoft Project wilt gaan exporteren, aan te bevelen om geen gebruik te maken van punttekens in uw WBS codes.

Elk niveau in de Microsoft Project outline code opzoektabel heeft een masker nodig dat ervoor gedefinieerd moet worden, om de opmaak van de items op elk niveau van de tabel te specificeren. Omdat alfanumerieke opmaak bij alle niveaus van de WBS gebruikt, wordt elk code masker een "iedere" lengte teken volgorde toegewezen bij het exporteren. Microsoft Project staat alleen maar niet-alfanumerieke tekens toe om als masker scheidingstekens te gebruiken. Wanneer er een alfanumeriek teken ingevoerd staat in het Pad scheidingsteken veld op het Opmaak tabblad van het Opties dialoogvenster in , wordt deze vervangen door een punt-teken bij het exporteren.

Unieke taak ID's

Microsoft Project ondersteunt de unieke taak ID's van 's niet, maar de unieke taak ID's van het project worden geëxporteerd naar een extra veld zodat ze niet verloren gaan wanneer bestanden weer terug geïmporteerd worden in .

 Bestandsverschillen bij het importeren van projecten met MPP of XML formaat

Let bij het importeren van een MPP of XML bestand naar op de volgende punten.

Nul taken

Microsoft Project ondersteunt geen balken op zich, echter, ze ondersteunt wel nul taken. Bij het importeren worden nul taken omgezet in lege balken.

Taken

Deadlines worden in Microsoft Project anders gedefinieerd dan vlaggen; in is een deadline een vlag. Als een taak zowel een deadline als een vlag heeft, wordt de deadline bij het importeren verwijderd.

Microsoft Project ondersteunt het concept van gesplitste taken: taken die gescheiden worden in meerdere segmenten met ruimtes ertussen. Gesplitste taken worden bij het importeren omgezet in meerdere taken op dezelfde regel met een link er tussen - met de bijbehorende positieve/negatieve wachttijd:

Microsoft Project

Mijlpalen

Microsoft Project ondersteunt slechts één type mijlpaal, waar er twee ondersteunt. Mijlpalen worden omgezet in startmijlpalen bij het importeren wanneer er meer uitgaande dan inkomende links zijn. In alle andere gevallen worden de mijlpalen omgezet in eindmijlpalen.

Links met een positieve/negatieve wachttijd

In Microsoft Project wordt elke positieve/negatieve wachttijd op links tussen taken die verschillende kalenders gebruiken altijd berekend met de kalender van de opvolgertaak.

Kosten

Microsoft Project 2007 gaat vergeleken met eerdere versies anders om met kosten. Wanneer u een project vanuit Microsoft Project 2007 importeert, worden alle in het project gecreëerde kosten geïmporteerd als verbruiksgoederen met als kosten de waarde nul.

Kosten tarieven

Resources in Microsoft Project kunnen maximaal vijf tarieftabellen hebben, met daarin kosten die kunnen veranderen in de tijd: tot vijfentwintig keer in totaal. Een tarief wordt bij het importeren voor elke tarieftabel aangemaakt van een resource, en elke toewijzing krijgt de juiste kosten en inkomsten tarief toegewezen. Als het tarief veranderd in de tijd, worden er bij het importeren meerdere kosten en inkomsten tarieven aangemaakt gemaakt, met namen die de datum aangeven waarop het tarief veranderd.

Als een toewijzing een tarief krijgt dat halverwege de toewijzing veranderd, wordt de toewijzing bij het importeren in tweeën gesplitst en krijgt ieder deel het bijbehorende kosten en inkomsten tarief.

Negatieve kosten en inkomsten

ondersteunt positieve en negatieve kosten, en positieve en negatieve inkomsten. Microsoft Project ondersteunt positieve en negatieve kosten, maar kent het concept inkomsten niet. U specificeert hoe negatieve kosten moeten worden behandeld tijdens het importeerproces. U kunt ervoor kiezen om alle negatieve kosten te converteren naar inkomsten, of om ze negatieve kosten te laten zijn in het resulterende bestand.

Overwerk

Overwerk in Microsoft Project verandert de beschikbaarheid per dag van een toewijzing en geeft de mogelijkheid om de hoeveelheid werk voor een dag aan te passen. Als overwerk wordt aangegeven bij een toewijzing in Microsoft Project, stijgt de toewijzing van de resource bij het importeren om er zorg voor te dragen dat de duur van de taak en de hoeveelheid werk van de toewijzing niet veranderd. In de bijbehorende kalender wordt een uitzondering aangemaakt voor het overwerk tijdens het importeren.

Codebibliotheken

Wanneer een project waarbij codebibliotheek items toegewezen aan taken en resources zijn geëxporteerd was naar Microsoft Project, wordt de hiërarchie van codes opnieuw gecreëerd wanneer het bestand zelf als project weer terug in wordt geïmporteerd. Wanneer het bestand geïmporteerd wordt in een bestaande samenvattingsgroep, worden de codebibliotheek gegevens zoveel mogelijk gematcht met de aanwezige codebibliotheek items om te voorkomen dat er meerdere codebibliotheek items met dezelfde naam worden gecreëerd.

Omdat codebibliotheek opmaakgegevens niet worden geëxporteerd, wordt er een set met vooraf gedefinieerde opmaak toegewezen aan de geïmporteerde codes. Deze opmaak zou niet kunnen matchen met de opmaak van de codes zoals die vóór het aanvankelijke exporteren naar Microsoft Project waren.

Work breakdown structure

Wanneer een project waarin een work breakdown structure gedefinieerd is geëxporteerd was naar Microsoft Project, dan wordt de work breakdown structure opnieuw gecreëerd wanneer het bestand terug wordt geïmporteerd in . Wanneer u een MPP of XML bestand in een kaart of samenvattingsgroep importeert in plaats van het als een geheel project te importeren, blijft de bestaande work breakdown structure van het project behouden en worden de WBS codes waar mogelijk toegewezen aan de geïmporteerde taken. Wanneer de WBS code van een geïmporteerde taak niet in de work breakdown structure aanwezig is, wordt deze er niet aan toegevoegd en wordt er geen WBS code toegewezen aan de geïmporteerde taak.

Creëren van Bibliotheek Verkenner objecten

Wanneer u een MPP of XML bestand in een kaart of samenvattingsgroep importeert in plaats van het als een geheel project te importeren, zouden er als gevolg van het importeren een aantal Bibliotheek Verkenner objecten, zoals kalenders, kostengroepen, kosten- en inkomstentarieven en resources, kunnen worden gecreëerd. Deze worden ofwel in de root van de relevante Bibliotheek Verkenner hiërarchieën of in mappen die de naam krijgen die u kunt specificeren tijdens het importeren.

Wanneer er in een samenvattingsgroep wordt geïmporteerd dan kan de standaardkalender niet worden aangepast en zal er ook niet geprobeerd worden om de geïmporteerde resources of kalenders te matchen met de al in het project aanwezig zijnde objecten. De standaard kalender van het Microsoft Project project wordt geïmporteerd en krijgt als naam "Standard". Iedere naamsclash met al aanwezige objecten wordt opgelost door de nieuw-geïmporteerde objecten te hernoemen (er wordt een aantal aanhalingstekens toegevoegd aan hun namen).

Extra velden en gebruikersvelden

Wanneer u een MPP of XML bestand in een kaart of samenvattingsgroep importeert in plaats van het als een geheel project te importeren, zullen de extra velden in het project waar mogelijk op naam gemapt worden aan de gebruikersvelden in het project. Wanneer er in het project aanvullende gebruikersvelden zouden moeten worden gecreëerd om er de extra veld informatie in op te slaan, verschijnt er een dialoogvenster dat u informeert of dit al dan niet mogelijk is; wanneer het mogelijk is, krijgt u de keuze om door te gaan met het creëren van het gebruikersveld, of om het project te importeren zonder aanvullende gebruikersvelden te creëren, of om het importeren helemaal te annuleren.
