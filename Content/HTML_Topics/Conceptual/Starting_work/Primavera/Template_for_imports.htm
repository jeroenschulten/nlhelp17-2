Een sjabloon toepassen wanneer er geïmporteerd wordt uit Microsoft Project of Primavera software

Wanneer u Microsoft Project en Primavera® projecten in importeert, kunt u een sjabloon toepassen op het project dat u aan het importeren bent. Het zou kunnen dat u een sjabloon wilt toe passen op het project zodat dit project wordt gepresenteerd op een voor u bekende wijze - bijvoorbeeld met een bepaalde set velden in de tabelkolommen.

U gebruikt dit dialoogvenster ook om aan te geven welk sjabloon - indien er eentje is - toegepast moet worden op het project dat u aan het importeren bent in . Het zou kunnen dat u een sjabloon wilt toepassen op het project om te zorgen dat wanneer het project geopend wordt in , deze op een voor u bekende wijze wordt gepresenteerd - bijvoorbeeld, met een bepaalde set velden weergegeven in tabelkolommen. Een klein aantal sjablonen wordt meegegeven met , maar u kunt desgewenst zelf extra sjablonen aanmaken.

Bestandsformaten waarop u een sjabloon kunt toepassen tijdens het importeren
  • MPP - Microsoft Project haar eigen bestandsformaat.
  • XML - wordt gebruikt bij Primavera P6®; en bij Microsoft Project vanaf versie 2002 en later.
  • XER - wordt gebruikt bij Primavera P6.
  • DIR/STX - wordt gebruikt door Primavera SureTrak®.

Een klein aantal sjablonen wordt meegegeven met , maar u kunt desgewenst zelf extra sjablonen aanmaken.

Om een sjabloon te creëren voor gebruik importeren van projecten:

  1. Creëer een nieuw project in .
  2. Maak één of meer views een view in het project en configureer deze om de informatie te tonen op de wijze waarin u wilt dat de projecten die u importeert verschijnen. Bijvoorbeeld, u zou een aantal views kunnen aanmaken dieeen view kunnen aanmaken die bepaalde velden in de tabelkolommen tonentoont en waarin de balkkaart op een bepaalde manier is opgemaakt.
  3. Het zou kunnen dat u ook items binnen de Bibliotheek Verkenner wilt aanmaken in het sjabloon; Wanneer dat het geval is worden deze onderdeel van alle geïmporteerde projecten waarop het sjabloon is toegepast.
  4. Sla het project op in de map die staat gespecificeerd in het Sjabloon-project pad op het tabblad Bestandslocaties van het Opties dialoogvenster.

Om een sjabloon aan een project toe te voegen tijdens het importeren ervan, selecteer het gewenste sjabloon in het Gebruik sjabloon veld op het dialoogvenster dat verschijnt tijdens het importeerproces. Wanneer u geen sjabloon op een project toepast bij het importeren, zal het project openen met een standaard balkkaart view met een standaard set tabelkolommen.

Opmerkingen over project importsjablonen
  • Wanneer u een sjabloon toepast op een project bij het importeren, zullen alle Bibliotheek Verkenner items in het sjabloon onderdeel gaan uitmaken van het geïmporteerde project.
  • De standaard kalender in een sjabloon wordt overschreven door de standaard kalender in de projecten die u aan het importeren wanneer het sjabloon wordt toegepast. Dit betekent dat wanneer u views in een sjabloon configureert om niet-werktijd op te vouwen en/of met schaduw te tonen volgens de standaard kalender van het sjabloon, de views in het geïmporteerde project niet-werktijd opvouwen en/of schaduw zullen tonen volgens de correcte standaard kalender - d.w.z. de standaard kalender in het project dat wordt geïmporteerd.

    De 'standaard' kalender in Primavera P6 projecten is degene die gelabeld is met de 'standaard_vlag'; in Primavera SureTrak projecten is de eerste kalender die het importeerproces tegenkomt de 'standaard' kalender.
  • Wanneer het sjabloon is geconfigureerd om kostengroepen of codebibliotheken te bevatten, wordt de standaard kostengroep en codebibliotheek in het sjabloon overschreven door de eerste kostengroep en codebibliotheek in het project dat u aan het importeren bent wanneer het sjabloon wordt toegepast.
