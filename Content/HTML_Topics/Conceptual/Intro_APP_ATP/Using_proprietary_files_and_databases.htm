's eigen bestandsformaat

Behalve als u gebruik maakt van Enterprise, worden projecten opgeslagen in 's eigen bestandsformaat met de extensie .PP, bijvoorbeeld PROJECT.PP. U selecteert het bestandsformaat dat u wilt gebruiken via het Opslaan als type veld op het Opslaan als dialoogvenster.

Als u als een aan een Enterprise server verbonden client gebruikt, kunt u uw projecten ook opslaan in een database op de server (een server-mounted database of datasource), hetgeen een Microsoft SQL Server® bestandsformaat is. Raadpleeg de Help-tekst bij Serveradmin voor meer informatie over het opzetten van datasources.

U kunt projecten die zijn opgeslagen in een database op de server alleen openen als u in een Enterprise-omgeving werkt.

Projecten opslaan in een standalone omgeving

Als u projecten opslaat in een standalone-omgeving, wordt elk project opgeslagen in een apart bestand. Elk .PP bestand bevat één project. Elk project kan meerdere subprojecten of subkaarten bevatten:

Sleutel:

- Lokaal .PP

- Project (programma)

- Subkaart

Projecten opslaan in een Enterprise-omgeving

Wanneer u projecten opslaat in en u bent verbonden met een Enterprise server, kunt u elk project lokaal opslaan in een apart .PP bestand, of in een database of datasource van de server. Elke database van de server bevat één of meer groepsprojecten, die op hun beurt weer één of meer projecten kunnen bevatten. Elk project kan meerdere subprojecten of subkaarten bevatten:

Sleutel:

- Server-mounted database of datasource

- Groepsproject

- Project (programma)

- Subkaart

Bibliotheekobjecten, zoals resources, kalenders en codes, worden gedeeld door alle projecten in het groepsproject. Een resource dat u bijvoorbeeld hebt gemaakt voor een bepaald project, kan door elk ander project in hetzelfde groepsproject worden gebruikt.

De systeembeheerder maakt datasources met de toepassing Serveradmin. Als u projecten maakt die bibliotheekobjecten moeten delen met andere projecten, zorg er dan voor dat het project wordt opgeslagen in dezelfde datasource.
