Dialoogvenster Client/Server communicatie details

U maakt gebruik van het dialoogvenster Client/Server communicatie details om de methode te specificeren waarvan uw client gebruik zou moeten maken om verbinding te leggen met de server, of servers, waarop projecten zijn gelokaliseerd.

De manier waarop u dit doet hangt af of uw implementatie van Enterprise een Windows-gebaseerde of een web-gebaseerde bevat:

  • Als uw implementatie van Enterprise een Windows-gebaseerde omvat, moeten de serverlocatiegegevens overeenkomen met die gegevens die gespecificeerd zijn in . Bij dit type implementatie kunnen -clients verbinding maken met servers via één van de twee methoden, en als een client verbinding maakt met meer dan één server, kan deze een andere methode gebruiken om met elke server te communiceren. U moet alle communicatiemethoden specificeren die door een client worden gebruikt om verbinding te maken met zijn servers.
  • Als uw implementatie van Enterprise een webgebaseerde omvat, omvatten de serverlocatiegegevens de servernaam van de servermachine waarop is geïnstalleerd, vervolgens een dubbele punt en vervolgens een poortnummer - standaard 5400.
Hoe toegang te krijgen tot het Client/Server Communicatie Details dialoogvenster

Om toegang te krijgen tot het Client/Server Communicatie Details dialoogvenster, klik op het Bestand tabblad, klik op Openen, klik op Enterprise, en klik vervolgens op Server Locator.

 Verklaring van de velden op het dialoogvenster
Veld Omschrijving
Name Server

Enterprise implementaties met een Windows-gebaseerde

 

Schakel dit selectievakje in wanneer u gebruik maakt van de Name Server communicatie methode, via de applicatie Name Server. Wanneer u een Enterprise server start, communiceert deze met de Name Server, welke alle details van alle op dat moment draaiende Enterprise servers beheert. De Name Server publiceert vervolgens de details van de Enterprise servers naar de clients, zonder dat de clients naar elke afzonderlijke server hoeven te verbinden. De Name Server moet ten allen tijde draaien om clients en servers er naar te kunnen verbinden.

 

U moet deze server locatie methode gebruiken wanneer clients via het Internet verbinding moeten maken met een server, aangezien het de enige server locatie methode is die Internet-capabel is.

 

Voer het IP adres of de hostnaam in van de computer waarop de Name Server draait in het veld rechts van het selectievakje.

 

Enterprise implementaties met een web-gebaseerde

 

Schakel dit selectievakje in en voer de servernaam van de server machine waarop Enterprise geïnstalleerd is, gevolgd door een dubbele punt, en een poortnummer - standaard is dat 5400 - bijvoorbeeld PROJECTSERVER:5400.
Gedeelde Map

Schakel dit selectievakje in wanneer uw Enterprise implementatie een Windows-gebaseerde heeft en u gebruik maakt van de Gedeelde Map server locatie methode . Dan zal de client, servers lokaliseren die gebruik maken van een gedeeld bestand, dat wordt opgeslagen in de map die u specificeert in dit dialoogvenster. De clients gebruiken de bestanden die in de folder gezet zijn om te bepalen welke servers op dit moment beschikbaar zijn.

 

Voer het station en het pad van de map die u gedeeld heeft in het veld aan de rechterkant van het selectievakje. U kunt het pad rechtstreeks opgeven (bijvoorbeeld //servernaam/mapnaam/) of gebruik maken van een mapping naar een netwerk station (bijvoorbeeld T:/mapnaam/).
... een client/server communicatie opzetten via de Name Server methode (Windows-gebaseerde )?
  1. Schakel het selectievakje Name Server in.
  2. Voer het IP adres of de hostnaam in van de computer waarop de Name Server draait in het veld rechts van het selectievakje.
 ... een client/server communicatie opzetten via de Gedeelde Map server locatie methode (web-gebaseerde ?
  1. Schakel het selectievakje Gedeelde Map in.
  2. Voer het station en het pad van de map die u gedeeld heeft in het veld aan de rechterkant van het selectievakje. U kunt het pad rechtstreeks opgeven (bijvoorbeeld //servernaam/mapnaam/) of gebruik maken van een mapping naar een netwerk station (bijvoorbeeld T:/mapnaam/).
 ... een client/server communicatie opzetten in een Enterprise implementatie met een web-gebaseerde ?
  1. Schakel het selectievakje Name Server in.
  2. Voer de servernaam van de server machine waarop Enterprise geïnstalleerd is, gevolgd door een dubbele punt, en een poortnummer - standaard is dat 5400 - bijvoorbeeld PROJECTSERVER:5400.
