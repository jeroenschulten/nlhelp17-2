Dialoogvenster Security groep Eigenschappen - tabblad Algemeen

U gebruikt het Algemeen tabblad van het Security groep Eigenschappen dialoogvenster om de naam, en de toegangsrechten die leden van de security groep hebben over bepaalde aspecten van het project, te bewerken.

U kunt u de leden van de security groep speciale rechten geven voor balken en taken met een bepaalde code, uit een gespecificeerde "security bibliotheek", als aanvulling op reeds gedefinieerde toegangsrechten die u al via Kaart Toegangsrechten tabblad gedefinieerd heeft.

Verklaring van de velden op het dialoogvenster
Veld Omschrijving
Naam Voer de naam van de security groep in.
Rechten bibliotheek

Om toegangsrechten voor een bepaalde code of codebibliotheek toe te wijzen (als aanvulling op kaart toegangsrechten), selecteert u de betreffende codebibliotheek in dit veld. U kunt dan de toegangsrechten voor een bepaalde code of voor de hele bibliotheek specificeren op het Kaart Toegangsrechten tabblad.

 

Bijvoorbeeld, u heeft een codebibliotheek genaamd Mensen die een code bevat voor iedere medewerker in het bedrijf, zodat u elke gebruiker volledige toegang zou kunnen geven tot balken, taken of toewijzingen die hun code hebben. Deze rechten komen bovenop de toegangsrechten die per kaart zijn ingesteld. Een gebruiker kan dus alleen-lezen rechten hebben voor een kaart, maar volledige toegang tot balken, taken of toewijzingen met zijn code binnen de kaart.

 

 

Wanneer u een codebibliotheek selecteert in dit veld, moet u het Bibliotheek Toegangsrechten tabblad gebruiken om de bibliotheek alleen-lezen te maken, om de leden van de security groep te verhinderen om de toegangsrechten te wijzigen voor de bibliotheek.

Gebruiker Toegang

Klik op het juiste keuzerondje om de toegangsrechten op te geven die leden van deze security groep moeten hebben voor gebruikers.

 

 

Specificeren van de toegangsrechten over security groepen en gebruikersrecords.

Publiceer

Schakel dit selectievakje in om het de leden van de security groep toe te staan details van resourcetoewijzingen te publiceren naar Microsoft Outlook® en via SMTP e-mail, of schakel dit selectievakje uit om te voorkomen dat zij dat kunnen doen.

 

 

Specificeren of de leden van de security groep details van resourcetoewijzingen kunnen publiceren naar Microsoft Outlook® en via SMTP e-mail

Archiveer/restore

Schakel dit selectievakje in om de leden van de security groep het recht te geven gegevens te archiveren van en terug te zetten met de Archive Manager applicatie of schakel het selectievakje uit om de leden van de security groep te beletten dit te kunnen doen.

 

Leden van de security groep kunnen alleen gegevens archiveren van kaarten en samenvattingsgroepen waarvoor zij Lezen rechten hebben. Dit is gedefinieerd op het Kaart Toegangsrechten tabblad van het dialoogvenster. U kunt alleen gegevens archiveren en terugzetten wanneer u gebruik maakt van Enterprise.

 

Specificeren of ze gegevens mogen archiveren en terugzetten

Taak Pool toegang

Klik op het juiste keuzerondje om te specificeren dat de leden van de security groep geen toegang hebben tot taak pools.

Bewaar project macro's

Schakel dit selectievakje in om de leden van de security groep toe te staan project macro's op te slaan (en daarvoor te overschrijven) , of schakel het selectievakje uit wanneer u niet wilt dat de leden van de security groep dit doet. Dit kan nuttig zijn als er bepaalde bedrijfsbrede project macro's zijn die de gebruikers niet per ongeluk zouden mogen overschrijven.

Bewerken tijdlijn

Schakel dit selectievakje in om aan te geven dat leden van deze security groep taken aan een projecttijdlijn kunnen toevoegen, taken uit een tijdlijn kunnen verwijderen en een tijdlijn kunnen opmaken, of schakel het selectievakje uit als u niet wilt dat ze dit kunnen doen.

 

Als u dit selectievakje uitschakelt, kunnen leden van deze security groep nog steeds de projecttijdlijn bekijken, de tijdlijn exporteren, 'naar' een taak in de tijdlijn gaan en de eigenschappen van een taak in de tijdlijn bekijken.
